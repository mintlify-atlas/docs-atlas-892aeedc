## Prerequisites

Before cloning the repository, make sure you have the following installed:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node">
    Version 18 or higher. Required by Next.js 14 and the full dependency tree.
  </Card>
  <Card title="Package Manager" icon="box">
    `pnpm` is preferred — the repository ships a `pnpm-lock.yaml`. `npm` and `yarn` work too.
  </Card>
</CardGroup>

<Tip>
  Install pnpm globally with `npm install -g pnpm` if you don't have it yet.
</Tip>

## Setup Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/v0-cidteswebsite.git
    cd v0-cidteswebsite
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash pnpm
      pnpm install
      ```

      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```
    </CodeGroup>

    This installs all runtime and dev dependencies declared in `package.json`, including Next.js 14, Tailwind CSS, Radix UI primitives, Recharts, Three.js, and more.
  </Step>

  <Step title="Run the development server">
    <CodeGroup>
      ```bash pnpm
      pnpm dev
      ```

      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```
    </CodeGroup>

    Next.js starts the dev server with Turbopack/hot reload enabled.
  </Step>

  <Step title="Open the app">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser. The home page (`/`) should load with the CIDTES navigation header and the interactive circular service diagram.
  </Step>
</Steps>

## Available Scripts

All scripts are defined in `package.json`:

```json
{
  "scripts": {
    "dev":   "next dev",
    "build": "next build",
    "start": "next start",
    "lint":  "next lint"
  }
}
```

| Script | Command | Purpose |
|---|---|---|
| `dev` | `pnpm dev` | Start local development server with HMR |
| `build` | `pnpm build` | Create an optimised production build |
| `start` | `pnpm start` | Serve the production build locally |
| `lint` | `pnpm lint` | Run ESLint via `next lint` |

<Note>
  The `next.config.mjs` sets `eslint.ignoreDuringBuilds: true` and `typescript.ignoreBuildErrors: true`, so `pnpm build` will not fail on lint or type errors. These should still be resolved during development.
</Note>
