## Directory Purposes

  <Card title="styles/" icon="paintbrush">

  </Card>

## App Router Route Structure

The project uses the **Next.js 14 App Router** (`app/` directory). Each folder with a `page.tsx` is a publicly accessible route:

| Route | File | Page |
|---|---|---|
| `/` | `app/page.tsx` | Home — about, commitments, service diagram |
| `/capacitacion` | `app/capacitacion/page.tsx` | Training services |
| `/certificacion` | `app/certificacion/page.tsx` | Certification processes |
| `/consultoria` | `app/consultoria/page.tsx` | Consulting services |
| `/investigacion-consultoria` | `app/investigacion-consultoria/page.tsx` | Research & consulting |
| `/cursos` | `app/cursos/page.tsx` | Course catalogue |

<Info>
  All pages are `"use client"` components because they rely on React state for interactive UI elements such as carousels, the circular diagram, and the certification wheel.
</Info>

## Root Layout

`app/layout.tsx` sets global metadata and injects the **Geist** font family (both Sans and Mono variants from the `geist` package):

```typescript
import type { Metadata } from "next"
import { GeistSans } from "geist/font/sans"
import { GeistMono } from "geist/font/mono"

export const metadata: Metadata = {
  title: "CIDTES",
  description: "Created by CIDTES",
  icons: { icon: "/images/logo-simple.png" },
}
```

## Key Configuration Files

<Tabs>
  <Tab title="next.config.mjs">
    ```javascript
    const nextConfig = {
      eslint: {
        ignoreDuringBuilds: true,  // ESLint errors won't fail CI builds
      },
      typescript: {
        ignoreBuildErrors: true,   // TS errors won't fail CI builds
      },
      images: {
        unoptimized: true,         // Serves images as-is (no Next.js optimisation)
      },
    }
    ```

    `images.unoptimized: true` is set because the site is deployed to Vercel's static hosting tier where the Image Optimization API is not required.
  </Tab>
  <Tab title="tailwind.config.ts">
    Tailwind v3 with `darkMode: ["class"]`. Extends the default theme with:

    - **CSS variable-based colours** — `background`, `foreground`, `card`, `primary`, `secondary`, `muted`, `accent`, `destructive`, `border`, `input`, `ring`, `chart-1…5`, and `sidebar-*` tokens
    - **Custom border radius** — `lg`, `md`, `sm` mapped to `--radius` CSS variable
    - **Custom keyframes** — `accordion-down` and `accordion-up` for Radix UI accordion animations
    - **Plugin** — `tailwindcss-animate`

    Content globs scan `pages/**`, `components/**`, `app/**`, and root-level files.
  </Tab>
  <Tab title="tsconfig.json">
    ```json
    {
      "compilerOptions": {
        "target": "ES6",
        "strict": true,
        "module": "esnext",
        "moduleResolution": "bundler",
        "jsx": "preserve",
        "incremental": true,
        "paths": {
          "@/*": ["./*"]
        }
      }
    }
    ```

    The `@/*` path alias maps to the project root, so `import { NavHeader } from "@/components/nav-header"` resolves to `./components/nav-header.tsx`.
  </Tab>
</Tabs>
